# Executive Project Tracking Dashboard - โรงเรียนครุศาสตร์

ระบบแดชบอร์ดแสดงผลการติดตามแผนงานโครงการสำหรับผู้บริหาร โรงเรียนครุศาสตร์ แสดงผลข้อมูลโครงการ งบประมาณ และความคืบหน้าแบบ Real-time โดยดึงข้อมูลโดยตรงจาก Google Sheets ทุกครั้งที่มีการเปิดหน้าเว็บ

## 🌟 คุณสมบัติเด่น (Features)
- **ข้อมูลแบบ Real-time:** ดึงข้อมูลสดจาก Google Sheet ทุกครั้งที่เปิดหน้าเว็บหรือกดปุ่มรีเฟรช ปราศจากความล่าช้าของข้อมูล
- **การประมวลผลสถิติผู้บริหาร (KPI Cards):**
  - จำนวนโครงการทั้งหมด (แยกตามสถานะการดำเนินงาน)
  - สรุปงบประมาณรวมทั้งหมด
  - สรุปงบประมาณที่ใช้ไปแล้ว (พร้อมเปอร์เซ็นต์อัตราส่วนการใช้จ่าย)
  - สรุปงบประมาณคงเหลือ (พร้อมเปอร์เซ็นต์คงเหลือ)
  - อัตราความคืบหน้าเฉลี่ยของทุกโครงการ
- **ระบบกราฟข้อมูลระดับ Premium (Interactive Charts):**
  - กราฟแท่งเปรียบเทียบงบประมาณที่ตั้งไว้กับงบประมาณที่ใช้จริงแยกตามกลุ่มงาน
  - กราฟวงกลมแสดงสัดส่วนสถานะของโครงการทั้งหมด
- **ตารางข้อมูลโครงการแบบละเอียด (Interactive Table):**
  - ค้นหาข้อมูล (Search) จากชื่อโครงการ ผู้รับผิดชอบ หรือรหัสโครงการ
  - ตัวกรองข้อมูล (Filters) แยกตามกลุ่มงาน หรือตามสถานะดำเนินงาน
  - เมื่อคลิกเลือกแต่ละแถวจะแสดง Modal ข้อมูลวิเคราะห์โครงการนั้นแบบละเอียดพร้อมความคืบหน้า
- **ระบบสลับธีมสี (Light / Dark Mode):** รองรับการใช้งานทั้งแบบโหมดมืด (ค่าเริ่มต้น) และโหมดสว่าง

---

## 🛠️ โครงสร้างไฟล์
- `index.html` - โครงสร้างหน้าเว็บ แผงควบคุม และโครงสรุปของแดชบอร์ด
- `style.css` - ตกแต่งสไตล์แบบพรีเมียม (Glassmorphism), รองรับธีมมืด/สว่าง และรองรับการแสดงผลทุกหน้าจอ (Responsive)
- `app.js` - ตรรกะการประมวลผลดึงข้อมูลจาก Google Sheets และจัดการการเชื่อมโยงระบบควบคุมในหน้าเว็บ

## 📁 ระบบจัดการเอกสารจัดซื้อจัดจ้าง (Procurement Document System)

ระบบแดชบอร์ดรองรับการอัปโหลดและดาวน์โหลดเอกสารจัดซื้อจัดจ้างแยกตามโครงการใน 2 รูปแบบ:

### 1. การแนบเอกสารโลคอลผ่านหน้าเว็บ (Local Browser Storage)
- **วิธีการใช้งาน:** ผู้ใช้งานสามารถเปิด Modal ของโครงการใดก็ได้ จากนั้นลากและวางไฟล์ (Drag & Drop) หรือคลิกที่กล่องอัปโหลดเพื่อแนบไฟล์ (เช่น PDF, Word, Excel, รูปภาพ) สูงสุด 20MB ต่อไฟล์
- **การเก็บรักษาข้อมูล:** ข้อมูลไฟล์จะถูกเข้ารหัสและเก็บบันทึกไว้ในฐานข้อมูล **IndexedDB** ในเว็บเบราว์เซอร์ของคุณโดยอัตโนมัติ ซึ่งมีความปลอดภัยสูง จัดเก็บไฟล์ขนาดใหญ่ได้ และไม่สูญหายเมื่อรีเฟรชหน้าเว็บหรือปิดเบราว์เซอร์
- **ความเหมาะสม:** เหมาะสำหรับการบริหารจัดการเอกสารจากเครื่องของผู้บริหารโดยตรง ไม่ต้องผ่านเซิร์ฟเวอร์กลาง

### 2. การเชื่อมโยงลิงก์ผ่าน Google Sheets (Cloud Integration)
- **วิธีการใช้งาน:** ผู้ใช้สามารถเพิ่มคอลัมน์ชื่อ **"เอกสารจัดซื้อจัดจ้าง"** (หรือ **"ลิงก์เอกสาร"**) ใน Google Sheets ต้นทาง
- จากนั้นใส่ลิงก์แชร์เอกสารของ Google Drive หรือลิงก์คลาวด์อื่นๆ ในแถวโครงการที่ต้องการ
- เมื่อหน้าแดชบอร์ดโหลดข้อมูลเสร็จ ระบบจะแสดงปุ่มเปิดเอกสารคลาวด์ดังกล่าวเป็นไอคอนก้อนเมฆสีฟ้าในรายการเอกสารของโครงการนั้นทันที
- **ความเหมาะสม:** เหมาะสำหรับการแชร์ข้อมูลเอกสารร่วมกันเป็นทีมเพื่อให้ผู้บริหารทุกคนมองเห็นชุดเอกสารคลาวด์ชุดเดียวกัน

## 💻 วิธีการเปิดใช้งานในเครื่องโลคอล (Local run)
เนื่องจากระบบนี้เป็น Static Web Application ที่ประมวลผลฝั่งเบราว์เซอร์ของลูกค้า (Client-side) คุณสามารถเปิดใช้งานได้ง่ายๆ ด้วยวิธีใดวิธีหนึ่งดังนี้:
1. **ดับเบิลคลิกเปิดไฟล์ `index.html`** ในโฟลเดอร์นี้เพื่อรันผ่านเบราว์เซอร์ได้ทันที
2. หรือรันผ่าน Local Server เช่น ใช้ส่วนเสริม **Live Server** ใน VS Code หรือรันผ่านคำสั่ง Python:
   ```bash
   python -m http.server 8000
   ```
   แล้วเข้าใช้งานผ่านลิงก์ `http://localhost:8000`

## 🚀 วิธีการอัปโหลดและเผยแพร่บน GitHub Pages (Deployment Guide)
คุณสามารถนำแดชบอร์ดนี้เผยแพร่เป็นเว็บไซต์สาธารณะฟรีผ่าน **GitHub Pages** โดยทำตามขั้นตอนด้านล่างนี้:

### ขั้นตอนที่ 1: ตั้งค่า Git และ Commit ไฟล์ในเครื่องของคุณ
เปิดโปรแกรม Terminal หรือ PowerShell ในโฟลเดอร์โครงการนี้ แล้วรันคำสั่ง:
```bash
# 1. เริ่มต้นระบบควบคุมเวอร์ชัน Git
git init

# 2. เพิ่มไฟล์ทั้งหมดเข้าสู่สเตจเพื่อเตรียม Commit
git add .

# 3. Commit ไฟล์ลงระบบพร้อมเขียนข้อความบันทึก
git commit -m "Initial commit: Education School Dashboard"

# 4. เปลี่ยนชื่อสาขาหลักเป็น main
git branch -M main
```

### ขั้นตอนที่ 2: สร้าง Repository บน GitHub และ Push โค้ด
1. เข้าไปที่เว็บไซต์ [GitHub](https://github.com/) และลงชื่อเข้าใช้งาน
2. คลิกปุ่ม **New** เพื่อสร้าง Repository ใหม่:
   - ตั้งชื่อ Repository เช่น `education-project-dashboard`
   - ตั้งค่าเป็น **Public** (สาธารณะ)
   - ไม่ต้องเลือกเพิ่ม README, .gitignore หรือ License
   - คลิกปุ่ม **Create repository**
3. คัดลอกคำสั่งเชื่อมโยง remote ในหน้านั้นมารันต่อใน Terminal ของคุณ (แทนที่ `<USERNAME>` ด้วยชื่อบัญชี GitHub ของคุณ):
   ```bash
   # เชื่อมโยงกับรีโพสิทอรีบน GitHub
   git remote add origin https://github.com/<USERNAME>/education-project-dashboard.git
   
   # ส่งไฟล์โค้ดทั้งหมดขึ้น GitHub
   git push -u origin main
   ```

### ขั้นตอนที่ 3: เปิดใช้งาน GitHub Pages
1. ไปหน้า Repository ของคุณบนเว็บไซต์ GitHub
2. คลิกที่เมนู **Settings** (รูปฟันเฟืองด้านบน)
3. เมนูด้านซ้าย เลือกหัวข้อ **Pages**
4. ในส่วนของ **Build and deployment**:
   - **Source:** เลือกเป็น `Deploy from a branch`
   - **Branch:** เลือกสาขา `main` และโฟลเดอร์ `/ (root)`
   - คลิกปุ่ม **Save**
5. รอระบบ GitHub Actions ประมวลผลประมาณ 1-2 นาที แฟลตฟอร์มจะแจ้ง URL สาธารณะของแดชบอร์ดคุณด้านบนสุด (เช่น `https://<USERNAME>.github.io/education-project-dashboard/`)
6. คุณสามารถนำลิงก์ดังกล่าวแชร์ให้ผู้บริหารเข้าชมได้ทันทีผ่านคอมพิวเตอร์ แท็บเล็ต หรือสมาร์ทโฟน
